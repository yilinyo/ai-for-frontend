## Why

当前系统已经能管理简历、岗位、投递记录和面试进展，但面试中被问到的问题仍主要沉淀在自由文本小记里，难以按标签、难度、掌握状态和真实提问频率复习。新增面试题库可以把真实面试反馈转化为可管理、可关联、可复习的个人知识资产，补齐从面试复盘到面试前训练的闭环。

## What Changes

- 新增独立一级模块“面试题库”，左侧菜单下包含“题库列表”和“闪卡复习”两个二级入口，内容区不再使用 tab 占位，但二者作为不同能力分别建模和验收。
- 支持面试题的新增、编辑、删除、查看详情，以及按关键词、标签、难度、题型、掌握状态、收藏状态等条件筛选。
- 支持点击题库列表项任意位置进入题目详情，操作按钮仍保留各自行为并避免误触详情跳转。
- 支持题目内容和答案解析使用 Markdown 格式，允许在编辑时粘贴图片附件并以 Markdown 图片形式显示。
- 支持题目答案解析、标签分类、难度、题型、掌握状态、收藏状态、创建时间、修改时间、最近复习时间和真实提问次数；题库列表为保证表格展示只显示创建时间，详情页同时显示创建时间和修改时间。
- 支持将题目关联到投递记录或具体面试进展，形成“面试提问记录”，记录公司、岗位、面试阶段、当时问法和回答表现等上下文，用于统计真实面试提问频率。
- 支持在题目详情页通过双击题目内容或答案解析快速进入编辑，除显式“编辑题目”按钮外提供更轻量的修改入口。
- 支持从投递记录或面试进展复盘中沉淀题目，形成“面试记录 -> 题库 -> 复习”的闭环。
- 支持闪卡复习模式，包括随机复习、标签专项、难度专项、收藏优先、薄弱优先和高频优先。
- 闪卡采用卡片式交互，点击或按钮触发 3D 翻转显示答案，支持左右滑动切换题目，并在自评后记录不会、模糊或掌握结果。
- 增加 Mock API、模型、状态管理、路由和视图，以支撑面试题库、面试提问记录和闪卡复习记录。

## Capabilities

### New Capabilities

- `interview-question-library`: 管理面试题库列表、题目分类与收藏、删除确认、题目详情、真实面试提问记录，以及从投递或面试进展沉淀题目。
- `interview-flashcard-review`: 管理闪卡复习入口、抽题模式、卡片翻转与滑动交互、自评结果记录，以及题目掌握状态和最近复习时间更新。

### Modified Capabilities

- 无。

## Impact

- 新增模型：面试题、面试提问记录、闪卡复习记录。
- 新增 API 与 Mock 数据：题目列表与详情、题目 CRUD、收藏状态切换、面试提问记录、闪卡抽题、复习结果保存。
- 新增 Vuex 模块：维护题库列表、当前题目、面试提问记录、闪卡队列和复习状态。
- 新增路由与页面：面试题库一级菜单、题库列表二级入口、题目详情页、闪卡复习二级入口。
- 影响投递记录详情或面试进展相关页面：增加从面试复盘沉淀题目或关联已有题目的入口。
- 视觉与交互需与现有 Vue 2 + TypeScript + Element UI 后台风格保持一致，同时补充闪卡翻转和滑动动画体验。
