## ADDED Requirements

### Requirement: 面试题库必须作为独立模块提供题库列表
系统 MUST 在左侧导航提供独立一级入口“面试题库”，并在其下提供“题库列表”和“闪卡复习”两个二级入口，其中题库列表能力负责题目资产管理。

#### Scenario: 用户进入题库列表
- **WHEN** 用户点击左侧导航中的“面试题库”
- **THEN** 系统展开或展示“题库列表”和“闪卡复习”两个二级菜单
- **AND** 默认进入“题库列表”页面
- **AND** 内容区域不再额外展示 tab 切换控件

### Requirement: 题库列表必须支持题目管理和筛选
系统 MUST 允许用户在题库列表中新增、编辑、查看、删除面试题，并支持按关键词、标签、难度、题型、掌握状态和收藏状态筛选题目。

#### Scenario: 用户筛选题库列表
- **WHEN** 用户在题库列表中输入关键词并选择标签、难度、题型、掌握状态或收藏状态
- **THEN** 系统按筛选条件展示匹配的面试题
- **AND** 每条题目展示标题、标签、难度、题型、掌握状态、提问次数、收藏状态、创建时间和最近复习时间
- **AND** 题库列表不展示修改时间，以保证表格列宽和主要信息可读性

#### Scenario: 用户点击题库列表项进入详情
- **WHEN** 用户点击题库列表中某条题目的任意非操作区域
- **THEN** 系统进入该题目的详情页
- **AND** 用户点击收藏、编辑、关联提问或删除按钮时，只执行对应操作，不触发详情跳转

#### Scenario: 用户新增面试题
- **WHEN** 用户创建一条面试题并填写标题、题目内容、答案解析、难度、题型和掌握状态
- **THEN** 系统保存该题目
- **AND** 题目出现在题库列表中
- **AND** 题目内容和答案解析支持 Markdown 文本
- **AND** 用户在题目内容或答案解析中粘贴图片时，系统以 Markdown 图片形式插入并可在详情或闪卡中显示

### Requirement: 面试题必须支持星星收藏和删除确认
系统 MUST 允许用户在题库列表中直接收藏或取消收藏题目，并在删除题目前进行二次确认。

#### Scenario: 用户切换题目收藏状态
- **WHEN** 用户点击题库列表中某条题目的星星收藏按钮
- **THEN** 系统切换该题目的收藏状态
- **AND** 已收藏题目显示为实心星
- **AND** 未收藏题目显示为空心星

#### Scenario: 用户删除面试题
- **WHEN** 用户点击题库列表中某条题目的删除按钮
- **THEN** 系统展示删除确认提示
- **AND** 用户确认后系统删除该题目
- **AND** 如果该题目存在面试提问记录或复习记录，系统同步处理关联数据，避免悬空记录

### Requirement: 面试题必须支持 Markdown 答案解析和详情查看
系统 MUST 为每道面试题保存 Markdown 格式的题目内容和答案解析，并允许用户在详情页查看题目基础信息、完整题目、答案解析、面试提问记录和复习摘要。

#### Scenario: 用户查看题目详情
- **WHEN** 用户从题库列表打开某道题目的详情页
- **THEN** 系统展示题目标题、标签、难度、题型、掌握状态和收藏状态
- **AND** 系统以 Markdown 预览形式展示完整题目内容和答案解析
- **AND** 系统展示该题的真实面试提问记录和复习摘要
- **AND** 系统展示该题的创建时间和修改时间

#### Scenario: 用户双击详情内容快速编辑
- **WHEN** 用户在题目详情页双击题目内容或答案解析区域
- **THEN** 系统打开题目编辑弹窗
- **AND** 编辑弹窗带入当前题目的完整内容和答案解析
- **AND** 用户保存后系统更新该题目并刷新详情展示

### Requirement: 面试提问记录必须关联真实投递或面试进展
系统 MUST 支持将面试题关联到一条投递记录，并可选择关联到该投递记录下的某一轮面试进展，用于统计题目真实提问频率。

#### Scenario: 用户为题目关联投递记录
- **WHEN** 用户为某道题目创建面试提问记录并选择投递记录
- **THEN** 系统保存题目 ID、投递记录 ID、公司名称快照、岗位名称快照和提问时间
- **AND** 该题目的提问次数增加

#### Scenario: 用户为题目关联面试进展
- **WHEN** 用户在创建面试提问记录时选择某一轮面试进展
- **THEN** 系统保存面试进展 ID 和面试阶段快照
- **AND** 题目详情页展示该题被问于对应公司、岗位和面试阶段

### Requirement: 投递详情必须提供沉淀题目的入口
系统 MUST 在投递详情或面试进展复盘场景中提供入口，使用户可以将真实面试问题沉淀为新题目或关联到已有题目。

#### Scenario: 用户从面试进展沉淀新题目
- **WHEN** 用户在投递详情的面试进展区域选择沉淀题目
- **THEN** 系统允许用户创建新面试题
- **AND** 系统为新题目创建与当前投递记录或面试进展关联的面试提问记录

#### Scenario: 用户从面试进展关联已有题目
- **WHEN** 用户在投递详情的面试进展区域选择关联已有题目
- **THEN** 系统允许用户选择题库中的已有题目
- **AND** 系统为该题创建与当前投递记录或面试进展关联的面试提问记录
