## 为什么要做

当前系统的能力主要停留在简历仓库和简历版本管理。用户在导出简历并开始投递后，系统无法继续追踪“哪一版简历投给了哪家公司、哪个岗位”，也无法沉淀与该版本对应的面试进展、面试笔记和个人复盘评分。

这会导致产品在核心求职流程中断档。实际使用中，同一个简历版本往往会被投递到多个岗位，而每一次投递又可能进入多轮面试并产生结构化反馈。如果没有与简历版本强关联的投递记录，用户就无法回顾哪一版简历效果更好、不同岗位关注了哪些要求，也无法根据面试反馈继续优化下一版简历。

## 变更内容

- 在每个简历版本下新增投递记录管理能力，使一个简历版本可以关联多个岗位投递记录。
- 在投递记录中补充岗位级信息，包括公司名称、公司介绍、岗位名称、岗位要求、base、薪资范围、投递渠道、投递时间、当前状态和备注。
- 为每条投递记录增加面试进展管理，支持记录笔试、一面、二面、HR 面、Offer、拒绝等阶段历史。
- 为每条投递记录增加面试小记和总结字段，便于用户记录问题、表现、失误点和后续改进方向。
- 为每条投递记录增加个人评分字段，支持从简历匹配度、面试表现、岗位意向度和综合评价等维度进行主观打分。
- 在简历版本详情页展示该版本关联的投递记录，并提供单独的投递详情页用于维护时间线和复盘信息。

## 能力定义

### 新增能力

- `resume-application-tracking`：支持在单个简历版本下跟踪多个投递记录，并保留该版本对应的求职历史。
- `interview-progress-and-review`：支持围绕每条投递记录维护面试阶段进展、面试小记、总结和个人评价。

### 修改能力

- `resume-version-detail-management`：扩展简历版本详情页，使其可以展示、创建和跳转关联投递记录。

## 影响范围

- 影响数据模型：需要在 `src/models/` 中新增投递记录与面试进展相关模型
- 影响 API 与 Mock：需要在 `src/api/` 和 `mock/` 中新增投递记录和面试进展的接口与模拟数据
- 影响状态管理：需要新增投递记录和面试进展对应的 Vuex 模块
- 影响页面：需要扩展简历版本详情页，并新增投递创建、编辑、详情相关页面
- 影响路由：需要在 `src/router/modules/` 中补充投递相关路由
- 影响数据约束：简历版本存在关联投递记录时，不能被静默删除
- 当前没有真实后端，因此首版实现仍应基于 Mock 数据和前端闭环完成
