## 为什么要做

当前系统已经具备简历版本、投递记录和面试进展管理能力，但岗位信息仍然主要附着在投递记录中，缺少一个可独立沉淀、复用和维护的岗位库。对于真实求职场景来说，用户往往会先收藏和整理岗位，再决定是否投递；如果岗位数据只能在投递时录入，就会带来重复录入、历史岗位无法沉淀、来源链接丢失以及多次投递难以复用等问题。

本次变更需要新增岗位库模块，将岗位作为独立对象存储，并支持从岗位详情直接创建投递记录。同时，岗位录入需要兼容两种方式：手动创建，以及输入原始岗位链接后调用解析接口，由后端返回结构化 JSON，前端只负责消费解析结果。这样可以在不引入真实解析逻辑的情况下，先把前端流程和数据结构跑通。

## 变更内容

- 新增独立岗位库模块，用于统一管理岗位信息。
- 新增岗位列表页，作为与“简历管理”同级的一级导航入口，风格与当前简历仓库列表页保持一致，采用顶部搜索筛选 + 卡片列表 + 分页结构。
- 新增岗位详情页，采用“岗位基础信息与操作”合并区域，并展示该岗位关联的投递记录。
- 新增岗位创建与编辑能力，支持手动录入，以及通过岗位来源链接调用解析接口自动回填字段。
- 在投递记录创建流程中支持直接选择岗位库中的已有岗位，并自动带出岗位字段。
- 为投递记录增加 `jobPostingId` 关联关系，同时保留岗位快照字段，避免岗位库后续修改影响历史投递语义。

## 能力定义

### 新增能力

- `job-posting-library-management`：支持岗位库的创建、查看、编辑、删除、筛选和来源链接跳转。
- `job-posting-link-parse`：支持输入岗位来源链接后调用解析接口，获得结构化岗位数据并回填表单。
- `job-posting-to-application-linking`：支持从岗位库选择岗位并生成与简历版本关联的投递记录。

### 修改能力

- `resume-application-creation-flow`：扩展投递记录创建流程，使其可以直接关联岗位库数据。

## 影响范围

- 影响数据模型：新增岗位库实体，并补充投递记录与岗位库之间的关联字段
- 影响 API 与 Mock：新增岗位库 CRUD 接口与岗位链接解析接口
- 影响状态管理：新增岗位库 Vuex 模块
- 影响页面：新增岗位列表页、岗位详情页、岗位编辑弹窗或表单页
- 影响路由与导航：新增与“简历管理”同级的左侧一级导航入口“岗位列表”
- 影响投递记录：创建投递记录时需要支持选择已有岗位并自动带出岗位信息
- 当前阶段仍采用 Mock-first 方式实现，解析接口也先通过 Mock 返回结构化 JSON
