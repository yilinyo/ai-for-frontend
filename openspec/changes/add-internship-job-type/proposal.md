## 为什么要做

当前简历仓库的求职类型只支持“校招”和“社招”，但实际个人求职场景中，“实习”是一个独立且高频的求职类型。实习求职通常与校招、社招在投递节奏、目标岗位、简历内容侧重点和复盘方式上都有差异，如果继续强行归入校招或社招，会影响用户对简历仓库的分类管理和后续筛选。

本次变更需要在现有“校招 / 社招”基础上新增“实习”求职类型，并让简历仓库创建、编辑、列表筛选、卡片展示和详情展示都能一致识别该类型。

## 变更内容

- 在简历仓库求职类型枚举中新增 `internship` 类型，对应展示文案为“实习”。
- 在简历仓库创建和编辑表单中新增“实习”选项。
- 在简历仓库列表筛选中新增“实习”筛选项。
- 在简历仓库卡片和详情页中正确展示“实习”标签，并提供与既有类型一致的视觉区分。
- 更新 Mock 简历仓库数据和筛选逻辑，保证 `internship` 类型可以参与列表查询。
- 同步更新相关文档口径，将“求职类型：校招 / 社招”扩展为“校招 / 社招 / 实习”。

## 能力定义

### 修改能力

- `resume-repo-job-type-management`：简历仓库求职类型必须支持“校招 / 社招 / 实习”三种选项，并在创建、编辑、筛选、展示和 Mock 数据中保持一致。

## 影响范围

- 影响数据模型：`src/models/resume-repo.ts` 中的 `JobType` 枚举和注释。
- 影响 Mock 数据：`mock/resume-repos.ts` 的类型定义、样例数据和筛选参数。
- 影响页面：`src/views/resume-repo/list.vue` 的筛选项与表单选项，`src/views/resume-repo/detail.vue` 的类型展示。
- 影响组件：`src/components/ResumeRepoCard/index.vue` 的类型文案和标签样式。
- 影响文档：项目 PRD、项目概要中关于“校招 / 社招”的描述需要补充“实习”。
- 不影响投递记录、面试进展、岗位库的核心数据结构；若后续需要岗位库也支持岗位类型，应单独提出 change。
