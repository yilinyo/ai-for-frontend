# interview-flashcard-review Specification

## Purpose
Define the flashcard review capability for practicing interview questions from the question library.

## Requirements
### Requirement: 闪卡复习必须作为独立能力提供复习入口
系统 MUST 在面试题库一级菜单下提供“闪卡复习”二级入口，并将其作为独立于题库列表管理的复习训练能力。

#### Scenario: 用户进入闪卡复习
- **WHEN** 用户在左侧“面试题库”菜单下点击“闪卡复习”二级入口
- **THEN** 系统展示闪卡复习页面
- **AND** 页面展示复习模式选择和闪卡区域
- **AND** 内容区域不再额外展示 tab 切换控件

### Requirement: 闪卡复习必须支持多种抽题模式
系统 MUST 在闪卡复习页面中支持随机复习、标签专项、难度专项、收藏优先、薄弱优先和高频优先抽题。

#### Scenario: 用户按复习模式抽题
- **WHEN** 用户在闪卡复习页面中选择复习模式和抽题范围
- **THEN** 系统按所选模式生成闪卡题目队列
- **AND** 系统展示第一张题目卡片的正面

### Requirement: 闪卡必须支持翻转显示答案
系统 MUST 以卡片形式展示闪卡，初始展示题目正面，并在用户点击卡片或翻转按钮后通过翻转动画展示 Markdown 格式的答案解析。

#### Scenario: 用户翻转闪卡查看答案
- **WHEN** 用户点击闪卡或点击翻转按钮
- **THEN** 系统以 3D 翻转动画展示卡片背面
- **AND** 卡片背面以 Markdown 预览形式展示答案解析和自评按钮
- **AND** 翻转过程中正反面文字不会同时可见

### Requirement: 闪卡必须支持滑动切换题目
系统 MUST 允许用户通过左右滑动卡片或切换按钮查看上一题和下一题，并在切换到新题后恢复卡片正面状态。

#### Scenario: 用户滑动到下一张闪卡
- **WHEN** 用户向左滑动当前闪卡或点击下一题按钮
- **THEN** 当前卡片以位移、缩放或透明度过渡滑出
- **AND** 下一张卡片自然进入
- **AND** 新卡片展示题目正面

#### Scenario: 用户滑动到上一张闪卡
- **WHEN** 用户向右滑动当前闪卡或点击上一题按钮
- **THEN** 系统展示上一张闪卡
- **AND** 该卡片展示题目正面

### Requirement: 闪卡复习结果必须更新题目复习状态
系统 MUST 在用户完成闪卡自评后记录复习结果，并更新题目的掌握状态和最近复习时间。

#### Scenario: 用户提交闪卡自评
- **WHEN** 用户在卡片背面选择不会、模糊或掌握
- **THEN** 系统保存一条闪卡复习记录
- **AND** 系统更新该题目的掌握状态
- **AND** 系统更新该题目的最近复习时间

#### Scenario: 用户完成一组闪卡复习
- **WHEN** 用户完成当前闪卡队列中的所有题目
- **THEN** 系统展示本次复习概览
- **AND** 概览包含复习总数、不会数量、模糊数量和掌握数量
