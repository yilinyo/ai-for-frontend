# 简历管理系统 - 快速启动指南

> 目录结构已调整：前端工程位于仓库根目录的 `frontend/` 下。本文为历史快速启动文档，执行前端命令前请先 `cd frontend`；文中 `src/`、`mock/`、`vue.config.js` 等路径均相对于 `frontend/`。

## 🚀 快速开始

### 1. 启动开发服务器

在前端目录执行:

```bash
cd frontend
npm run serve
```

这个命令会**同时启动**:
- **前端开发服务器**: http://localhost:8080
- **Mock API服务器**: http://localhost:9528

### 2. 访问应用

打开浏览器访问: **http://localhost:8080**

### 3. 登录系统

使用测试账号登录:
```
用户名: admin
密码: 111111
```

### 4. 访问简历管理功能

登录后,在左侧导航栏点击 **"简历管理"** 菜单,即可使用以下功能:

- **简历仓库** - 管理简历仓库,创建、编辑、删除仓库
- **个人信息** - 管理个人信息(姓名、年龄、邮箱、电话、求职意向等)

## 🔧 常见问题

### Q1: Mock服务器没有启动怎么办?

如果前端启动成功但无法获取数据,检查Mock服务器是否正常运行:

```bash
# 单独启动Mock服务器
npm run mock
```

查看控制台是否输出:
```
Mock server started on port 9528!
简历管理系统 API 已注册!
```

### Q2: 端口被占用怎么办?

如果8080或9528端口被占用:

**修改前端端口**: 编辑 `vue.config.js` 中的 `devServer.port`
**修改Mock端口**: 编辑 `mock/mock-server.ts` 中的 `port` 变量

### Q3: 页面报404错误

确保你访问的路由路径正确,所有简历相关的路由都以 `/resume/` 开头。

### Q4: 无法登录

检查:
1. Mock服务器是否正常启动(端口9528)
2. 用户名和密码是否正确(admin / 111111)
3. 浏览器控制台是否有API请求错误

## 🔄 切换到真实后端

当后端API开发完成后:

1. **修改API基础地址**
   编辑 `src/utils/request.ts`:
   ```typescript
   const service = axios.create({
     baseURL: 'https://your-api-domain.com', // 改为真实API地址
     timeout: 5000
   })
   ```

2. **停止Mock服务器**
   只运行前端开发服务器,不启动Mock:
   ```bash
   # 使用vue-cli-service直接启动
   npx vue-cli-service serve
   ```

3. **前端代码无需任何修改**
   所有API调用会自动切换到真实后端!
